The GRAFT.TODAY API is a set of Next.js route handlers. All endpoints are authenticated via Clerk session tokens.

## Base URL

```
https://graft.today/api
```

In local development: `http://localhost:3000/api`

## Authentication

All endpoints require an active Clerk session. Requests are authenticated via the Clerk session cookie when called from the portal frontend, or via a Bearer token for server-to-server calls.

```bash
curl https://graft.today/api/leads \
  -H "Authorization: Bearer <clerk_session_token>"
```

Unauthenticated requests return `401 Unauthorized`. Requests where the session resolves to no Client record return `404 Not Found`.

## Tenant scoping

All data returned by the API is automatically scoped to the authenticated tenant. You cannot access another tenant's data via the public API. Platform owners have cross-tenant access via separate admin endpoints at `/api/admin/`.

## Response format

Successful responses return JSON. Errors return a JSON object with an `error` field:

```json
{ "error": "Unauthorized" }
```

## Pagination

Endpoints that support pagination accept these query parameters:

| Parameter | Default | Description |
|---|---|---|
| `page` | `1` | Page number (1-indexed) |
| `limit` | `25` | Records per page (max: `100`) |

Paginated responses wrap results in an envelope:

```json
{
  "data": [...],
  "total": 142,
  "page": 1,
  "limit": 25,
  "pages": 6
}
```

Omit `page` and `limit` to receive all records in a flat array (no envelope).

## Sorting

Endpoints that support sorting accept:

| Parameter | Description |
|---|---|
| `sort` | Field to sort by (endpoint-specific) |
| `order` | `asc` or `desc` (default: `desc`) |
