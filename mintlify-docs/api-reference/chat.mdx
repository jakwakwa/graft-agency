## Send a message

Streams a chat response. This endpoint is called by the widget iframe — it is not intended for direct client use outside the widget.

```
POST /api/chat
```

### Request body

```json
{
  "messages": [
    { "role": "user", "content": "What services do you offer?" }
  ],
  "clientId": "uuid-of-tenant"
}
```

| Field | Required | Description |
|---|---|---|
| `messages` | Yes | Array of `{ role, content }` objects in [AI SDK message format](https://sdk.vercel.ai/docs/foundations/messages) |
| `clientId` | Yes | The tenant whose agent config and knowledge base to use |

### Response

Returns a streaming response in the [Vercel AI SDK data stream format](https://sdk.vercel.ai/docs/ai-sdk-ui/stream-protocol). Use `useChat` from `ai/react` on the client side to consume it.

```typescript
import { useChat } from "ai/react";

const { messages, input, handleInputChange, handleSubmit } = useChat({
  api: "/api/chat",
  body: { clientId: "your-client-id" },
});
```

### Tool calls

The chat endpoint may invoke tools on the server. The client receives tool call deltas in the stream and the widget renders appropriate UI for each:

| Tool | Client action |
|---|---|
| `capture-lead` | Shows a lead capture form |
| `book-appointment` | Shows a booking form (Cal.com embed) |
| `check-availability` | Shows available time slots |
| `handoff-human` | Shows a handoff confirmation message |
| `search-knowledge` | No visible UI — result injected into context |

### Rate limiting

Chat sessions are rate-limited per session ID. Exceeded limits return `429 Too Many Requests`.

### Domain restriction

Requests from domains not in the tenant's `allowedDomains` list return `403 Forbidden`. This prevents the widget from being embedded on unauthorised sites.

<Info>
  `localhost` is always allowed regardless of the allowed domains list, so local development works without configuration.
</Info>
