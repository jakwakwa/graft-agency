Each GRAFT.TODAY tenant is a **Clerk Organisation**. When a user creates or joins an organisation, a webhook syncs it to a `Client` row in Postgres.

## How tenants are resolved

Every authenticated API route calls `requirePlatformAccess()` from `lib/auth/resolve-client.ts`:

```typescript
import { requirePlatformAccess } from "@/lib/auth/resolve-client";

export async function GET(req: Request) {
  const result = await requirePlatformAccess();
  if ("error" in result) {
    return Response.json({ error: result.error }, { status: result.status });
  }
  const { clientId } = result;

  // All queries must include clientId
  const leads = await prisma.lead.findMany({ where: { clientId } });
}
```

This function reads the Clerk session, looks up the `Client` by `clerkUserId` or `clerkOrganizationId`, and returns `{ clientId }`. It throws a 401 if no session exists and a 404 if no matching Client is found.

## Clerk webhook sync

The webhook at `app/api/webhooks/clerk/` handles these events:

| Event | Action |
|---|---|
| `organization.created` | Creates a new `Client` record |
| `organization.updated` | Updates `businessName`, `email`, etc. |
| `organization.deleted` | Soft-deletes the Client (`deletedAt`) |
| `user.created` | Creates a Client for individual users |

<Warning>
  Always validate the `svix-signature` header before processing Clerk webhook payloads. The handler uses `CLERK_WEBHOOK_SECRET` for verification.
</Warning>

## Access tiers

The `Client` model has two elevated-access flags:

```typescript
isPlatformOwner  Boolean  // full cross-tenant access
isReseller       Boolean  // intermediate access
```

Set these directly in the database for the relevant Client records. They are not manageable via the portal UI.

## Data isolation rules

Every Prisma query that touches tenant data **must** include `clientId` in the `where` clause. The only exceptions are platform-owner routes in `app/api/admin/`.

```typescript
// ✅ Correct — scoped to tenant
const conversations = await prisma.conversation.findMany({
  where: { clientId },
});

// ❌ Wrong — leaks data across tenants
const conversations = await prisma.conversation.findMany();
```

## Server Actions

Server Actions are public endpoints. Always verify the session inside the action body — middleware alone does not protect them:

```typescript
"use server";
import { requirePlatformAccess } from "@/lib/auth/resolve-client";

export async function updateAgentConfig(data: unknown) {
  const result = await requirePlatformAccess();
  if ("error" in result) throw new Error("Unauthorized");
  const { clientId } = result;
  // validate data, then mutate
}
```
