## Prerequisites

- [Bun](https://bun.sh) — the project uses Bun as the runtime and package manager throughout
- PostgreSQL database (local or [Neon](https://neon.tech))
- A [Clerk](https://clerk.com) account for authentication
- A [Paddle](https://paddle.com) sandbox account for billing

## Installation

<Steps>
  <Step title="Clone and install dependencies">
    ```bash
    git clone <your-repo-url>
    cd graft-today-agency
    bun install
    ```
  </Step>

  <Step title="Set up environment variables">
    Copy the example env file and fill in your values:

    ```bash
    cp .env.example .env.local
    ```

    At minimum, set these variables before the dev server will start:

    | Variable | Description |
    |---|---|
    | `DATABASE_URL` | PostgreSQL connection string |
    | `CLERK_SECRET_KEY` | Clerk backend API key |
    | `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk frontend key |
    | `PADDLE_API_KEY` | Paddle API key (sandbox) |
    | `PADDLE_WEBHOOK_SECRET` | Paddle webhook secret |

    See [Environment variables](/deployment/environment-variables) for the full list.
  </Step>

  <Step title="Run database migrations">
    ```bash
    bun run db:migrate:dev
    ```

    To seed the database with initial data:

    ```bash
    bun run db:seed
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    bun run dev
    ```

    The app is available at `http://localhost:3000`.
  </Step>

  <Step title="Start the Inngest dev server (optional)">
    Run this in a separate terminal to process background events locally:

    ```bash
    inngest dev
    ```

    <Info>
      Set `INNGEST_DEV=true` in your `.env.local` to point the app at the local Inngest server.
    </Info>
  </Step>
</Steps>

## Webhooks in local development

Several features depend on inbound webhooks (Clerk org sync, Paddle billing, Cal.com bookings). Use the `dev:tunnel` script to expose your local server via a [Pinggy](https://pinggy.io) tunnel:

```bash
bun run dev:tunnel
```

Set `WEBHOOK_PUBLIC_BASE_URL` in `.env.local` to the printed tunnel URL.

## Useful commands

```bash
bun run build           # Prisma generate + next build (catches compile-time errors)
bun run lint            # Biome check
bun run format          # Biome format --write
bun run test            # Vitest unit tests
bun run test:e2e        # Playwright E2E (Chromium)
bun run db:migrate:dev  # Create a new Prisma migration
```

<Warning>
  Always run `bun run build` — not just the IDE type-checker — to catch TypeScript and Prisma contract errors that only surface at compile time.
</Warning>
