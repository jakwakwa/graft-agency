Copy `.env.example` to `.env.local` for local development. In production, set these via `vercel env` or your deployment platform's secrets manager.

<Warning>
  Never commit `.env.local` or any file containing real secrets to git.
</Warning>

## Core

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `TEST_DATABASE_URL` | Dev | Separate database for Vitest tests |
| `PORT` | No | Dev server port (default: `3000`) |

## Authentication (Clerk)

| Variable | Required | Description |
|---|---|---|
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Yes | Clerk frontend publishable key |
| `CLERK_SECRET_KEY` | Yes | Clerk backend secret key |
| `CLERK_WEBHOOK_SECRET` | Yes | Signs inbound Clerk webhook payloads |
| `NEXT_PUBLIC_CLERK_SIGN_IN_URL` | No | Custom sign-in URL (default: `/sign-in`) |
| `NEXT_PUBLIC_CLERK_SIGN_UP_URL` | No | Custom sign-up URL (default: `/sign-up`) |

## Payments (Paddle)

| Variable | Required | Description |
|---|---|---|
| `PADDLE_ENVIRONMENT` | Yes | `sandbox` or `production` |
| `PADDLE_API_KEY` | Yes | Paddle API key |
| `PADDLE_WEBHOOK_SECRET` | Yes | Signs inbound Paddle webhook payloads |
| `NEXT_PUBLIC_PADDLE_CLIENT_TOKEN` | Yes | Client-side Paddle token |
| `PADDLE_PRODUCT_*` | Yes | Product IDs for each plan |
| `PADDLE_PRICE_*` | Yes | Price IDs for each plan |

## AI and design

| Variable | Required | Description |
|---|---|---|
| `GCP_PROJECT_ID` | Yes | Google Cloud project ID |
| `GCP_STITCH_SA_ACCOUNT_BASE64_KEY` | Yes | Base64-encoded Stitch service account key |
| `JULES_API_KEY` | Yes | Jules Labs API key for code generation |
| `GITHUB_TOKEN` | Yes | GitHub token for Jules template repo access |
| `GITHUB_TEMPLATE_REPO` | Yes | `owner/repo` of the Jules template |

## Deployment services

| Variable | Required | Description |
|---|---|---|
| `RENDER_API_KEY` | Yes | Render API key for deploying generated apps |
| `RENDER_OWNER_ID` | Yes | Render owner/team ID |
| `VERCEL_WEBHOOK_SECRET` | No | Signs Vercel deployment webhook payloads |

## Bookings (Cal.com)

| Variable | Required | Description |
|---|---|---|
| `CAL_COM_API_KEY` | Yes | Cal.com API key |
| `CAL_USER_USERNAME` | Yes | Default Cal.com username |
| `CAL_USER_EVENT_SLUG` | Yes | Default event type slug |

## Event processing (Inngest)

| Variable | Required | Description |
|---|---|---|
| `INNGEST_DEV` | Dev | Set to `true` to use local Inngest server |
| `INNGEST_ENV` | Prod | Target Inngest environment name |
| `INNGEST_POLLING_ENABLED` | No | Enable/disable polling mode |

## Tunnels and webhooks

| Variable | Required | Description |
|---|---|---|
| `WEBHOOK_PUBLIC_BASE_URL` | Dev | Pinggy tunnel URL for local webhook testing |

## Monitoring

| Variable | Required | Description |
|---|---|---|
| `NEXT_PUBLIC_DATADOG_APPLICATION_ID` | No | Datadog RUM application ID |
| `NEXT_PUBLIC_DATADOG_CLIENT_TOKEN` | No | Datadog RUM client token |

## Engagement pipeline flags

| Variable | Default | Description |
|---|---|---|
| `ENGAGEMENT_DRY_RUN` | `false` | Stub Render, Paddle, and Resend calls |
