GRAFT.TODAY receives webhooks from four external services. Each webhook handler verifies the request signature before processing.

## Webhook endpoints

| Service | Path | Purpose |
|---|---|---|
| Clerk | `/api/webhooks/clerk` | Org and user sync |
| Paddle | `/api/webhooks/paddle` | Subscription state sync |
| Cal.com | `/api/webhooks/cal` | Booking confirmations |
| Vercel | `/api/webhooks/vercel` | Deployment status updates |

## Local webhook testing

Webhooks require a public URL. Use the tunnel script to get one:

```bash
bun run dev:tunnel
```

Copy the printed URL and set it in your `.env.local`:

```bash
WEBHOOK_PUBLIC_BASE_URL=https://abc123.a.pinggy.io
```

Then register this URL in each service's webhook settings:

- **Clerk**: Dashboard → Webhooks → Add endpoint → `{URL}/api/webhooks/clerk`
- **Paddle**: Dashboard → Notifications → New endpoint → `{URL}/api/webhooks/paddle`
- **Cal.com**: Settings → Developer → Webhooks → `{URL}/api/webhooks/cal`
- **Vercel**: Project → Settings → Webhooks → `{URL}/api/webhooks/vercel`

## Clerk webhooks

The Clerk webhook syncs organisation and user lifecycle events to the `Client` table.

Required events to subscribe to:
- `organization.created`
- `organization.updated`
- `organization.deleted`
- `user.created`
- `user.updated`

Set `CLERK_WEBHOOK_SECRET` to the signing secret from the Clerk dashboard.

## Paddle webhooks

Paddle webhooks keep subscription state in sync with the `Client` table.

Required events:
- `subscription.created`
- `subscription.updated`
- `subscription.canceled`
- `transaction.completed`

Set `PADDLE_WEBHOOK_SECRET` to the secret from Paddle → Notifications → your endpoint.

<Info>
  The Paddle handler idempotently updates subscription fields, so replays are safe.
</Info>

## Cal.com webhooks

Cal.com webhooks update `Lead.calBookingUid` and trigger follow-up actions when bookings are confirmed or cancelled.

Set `CAL_COM_API_KEY` for outbound Cal.com API calls from the chat widget's `book-appointment` tool.

## Vercel webhooks

Vercel deployment webhooks update `ProductSpec` when an engagement pipeline deployment completes or fails.

Set `VERCEL_WEBHOOK_SECRET` to the secret from Vercel → Project → Settings → Webhooks.

## Signature verification

All handlers use the respective SDK's verification utility. Never skip signature verification — even from trusted sources.

```typescript
// Example: Paddle signature check (simplified)
import { Paddle } from "@paddle/paddle-node-sdk";

const paddle = new Paddle(process.env.PADDLE_API_KEY!);
const signature = req.headers.get("paddle-signature") ?? "";
const event = paddle.webhooks.unmarshal(body, process.env.PADDLE_WEBHOOK_SECRET!, signature);
```
