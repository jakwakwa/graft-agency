# Chat Widget Brain — Feature Plan (Gold Rush Step 01)

## Overview

Implement the inbound AI chat widget "Brain": a streaming API route using Vercel AI SDK v6 with Google Gemini, five Zod-typed tools, service layer, Conversation persistence, iframe widget delivery, and embed script.

---

## File Tree

### New Files to Create

#### Prisma / Database

- `prisma/migrations/<timestamp>_add_conversation_model/migration.sql` — Generated by `bun prisma migrate dev`

#### Service Layer (`lib/services/`)

- `lib/services/agent.service.ts` — AgentConfig lookup + knowledge base search
- `lib/services/conversation.service.ts` — Conversation CRUD (save, load by session)
- `lib/services/cal.service.ts` — Cal.com API v2 wrapper (getAvailability, createBooking)

- `lib/ai/tools/capture-lead.ts` — `captureLeadDetails` tool definition (Zod + execute)

- `app/api/chat/route.ts` — POST handler: The Brain (streamText + createUIMessageStream)

## Module Size Guardrails

- No file exceeds 150 lines
- Chat route handler is thin — delegates to services + tools
- Widget page splits into 3 client components (chat-widget, tool-status, chat-input)
- Each tool definition is its own file (< 40 lines each)
- Services are per-domain, not monolithic
